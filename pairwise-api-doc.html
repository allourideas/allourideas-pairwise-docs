About the API

The API conforms to the RESTful Ruby on Rails ActiveResource pattern. For more details on ActiveResource, please see http://api.rubyonrails.org/files/vendor/rails/activeresource/README.html.

Questions

Creating a new question

Client->"/questions": POST params (xml)
    note right of "Client": Authenticate using HTTP.  Requires a question parameter hash.
    note right of "/questions": We add the question to our database
    "/questions"-->Client: xml version of the newly created question


Retrieving a specific question

Client->"/questions/:id": GET (xml)
    note right of "Client": Authenticate using HTTP.
    note right of "/questions/:id": We get the question from our database
    "/questions/:id"-->Client: xml version of the question


Updating a question

Client->"/questions/:id": PUT (xml)
    note right of "Client": Authenticate using HTTP.  Requires a question parameter hash.
    note right of "/questions/:id": We activate the question so that it can be shown to users
    "/questions/:id"-->Client: xml version of the question


Activating a question

Client->"/questions/:id/activate": PUT (xml)
    note right of "Client": Authenticate using HTTP.
    note right of "/questions/:id/activate": We activate the question so that it can be shown to users
    "/questions/:id/activate"-->Client: xml version of the question


Suspending a question

Client->"/questions/:id/suspend": PUT (xml)
    note right of "Client": Authenticate using HTTP
    note right of "/questions/:id/suspend": We suspend the question so that it will not be shown to users
    "/questions/:id/suspend"-->Client: xml version of the question


Clicks

Creating a click

Client->"/clicks": POST params (xml)
    note right of "Client": Authenticate using HTTP. Required parameters are "what_was_clicked_on" and "sid"
    note right of "/clicks": We add the click to our database
    "/clicks"-->Client: xml version of the newly added click


Choices

Creating a choice

Client->"/questions/:id/choices": POST params (xml)
    note right of "Client": Authenticate using HTTP.  Requires a choice parameter hash.
    note right of "/questions/:id/choices": We add the choice to our database
    "/questions/:id/choices"-->Client: xml version of the newly added choice


Retrieving choices

Client->"/questions/:id/choices": GET (xml)
    note right of "Client": Authenticate using HTTP.
    note right of "/questions/:id/choices": We get the choices for the question from our database
    "/questions/:id/choices"-->Client: xml version of the choices


Retrieving a specific choice

Client->"/questions/:question_id/choices/:id": GET (xml)
    note right of "Client": Authenticate using HTTP.
    note right of "/questions/:question_id/choices/:id": We get the choice from our database
    "/questions/:question_id/choices/:id"-->Client: xml version of the choice


Updating a specific choice

Client->"/questions/:question_id/choices/:id": GET (xml)
    note right of "Client": Authenticate using HTTP.  Requires a choice parameter hash.
    note right of "/questions/:question_id/choices/:id": We get the choice from our database
    "/questions/:question_id/choices/:id"-->Client: son version of the choice


Activating a choice

Client->"/questions/:question_id/choices/:id/activate_from_abroad": PUT (xml)
    note right of "Client": Authenticate using HTTP.
    note right of "/questions/:question_id/choices/:id/activate_from_abroad": We activate the choice so that it can be shown to users.  In future versions, this method should be renamed to "activate."
    "/questions/:question_id/choices/:id/activate_from_abroad"-->Client: xml version of the choice


Suspending a choice

Client->"/questions/:question_id/choices/:id/update_from_abroad": PUT (json or xml)
    note right of "Client": Authenticate using HTTP.
    note right of "/questions/:question_id/choices/:id/update_from_abroad": We suspend the choice so that it will not be shown to users.  In future versions, this method should be renamed to "suspend" or "deactivate."
    "/questions/:question_id/choices/:id/update_from_abroad"-->Client: xml or json version of the choice


Prompts

Prompts are different combinations of choices that can be shown to users

Retrieving a specific prompt

Client->"/questions/:question_id/prompts/:id": GET (xml)
    note right of "Client": Authenticate using HTTP.
    note right of "/questions/:question_id/prompts/:id": We get the prompt from our database
    "/questions/:question_id/prompts/:id"-->Client: xml version of the prompt


Activating a prompt

Client->"/questions/:question_id/prompts/:id/activate": PUT (xml)
    note right of "Client": Authenticate using HTTP.
    note right of "/questions/:question_id/prompts/:id/activate": We activate the prompt so that it can be shown to users
    "/questions/:question_id/prompts/:id/activate"-->Client: xml version of the prompt


Suspending a prompt

Client->"/questions/:question_id/prompts/:id/suspend": PUT (xml)
    note right of "Client": Authenticate using HTTP.
    note right of "/questions/:question_id/prompts/:id/suspend": We suspend the prompt so that it will not be shown to users
    "/questions/:question_id/prompts/:id/suspend"-->Client: xml version of the prompt


Vote for the choice on the left of the prompt

Client->"/questions/:question_id/prompts/:id/vote_left": POST params (xml)
    note right of "Client": Authenticate using HTTP.  Requires an 'auto' parameter corresponding to the session id.
    note right of "/questions/:question_id/prompts/:id/vote_left": We add a vote for the choice on the left
    "/questions/:question_id/prompts/:id/vote_left"-->Client: xml version of a new prompt for the question


Vote for the choice on the right of the prompt

Client->"/questions/:question_id/prompts/:id/vote_right": POST params (xml)
    note right of "Client": Authenticate using HTTP.  Requires an 'auto' parameter corresponding to the session id.
    note right of "/questions/:question_id/prompts/:id/vote_right": We add a vote for the choice on the right
    "/questions/:question_id/prompts/:id/vote_right"-->Client: xml version of a new prompt for the question


Skip prompt

Client->"/questions/:question_id/prompts/:id/skip": POST params (xml)
    note right of "Client": Authenticate using HTTP
    note right of "/questions/:question_id/prompts/:id/skip": We skip the prompt
    "/questions/:question_id/prompts/:id/skip"-->Client: xml version of a new prompt for the question
